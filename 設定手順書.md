# 設定手順書：今週の良かったこと3つだけ日記

## ファイル構成

```
yokatta-koto/
├── index.html   ← フロントエンド（HTML構造）
├── style.css    ← スタイルシート
├── script.js    ← フロントエンドのロジック（GAS URLを設定）
└── コード.gs    ← Google Apps Script に貼り付けるバックエンド
```

---

## STEP 1：Googleスプレッドシートを準備する

1. [Google スプレッドシート](https://sheets.google.com) を開く
2. 「空白」をクリックして新しいスプレッドシートを作成
3. 名前を「良かったこと日記」などに変更（任意）
4. **スプレッドシートIDをメモする**
   - URLのこの部分が ID です：  
     `https://docs.google.com/spreadsheets/d/【ここがID】/edit`

> ⚠️ シートの作成や行追加は GAS が自動で行うため、手動での操作は不要です。

---

## STEP 2：Google Apps Script を設定・デプロイする

### 2-1. GASを開く

1. スプレッドシートのメニューから **「拡張機能」→「Apps Script」** をクリック
2. Apps Script エディタが開く

### 2-2. コードを貼り付ける

1. デフォルトの `コード.gs` の内容をすべて選択して削除
2. 同梱の **`コード.gs`** の中身をすべてコピーして貼り付ける
3. **SPREADSHEET_ID を自分のものに変更する**  
   ```javascript
   const SPREADSHEET_ID = "YOUR_SPREADSHEET_ID_HERE";
   // ↑ ここをSTEP 1でメモしたIDに書き換える
   ```
4. Ctrl+S（Mac: Cmd+S）で保存

### 2-3. デプロイする

1. 右上の **「デプロイ」→「新しいデプロイ」** をクリック
2. 「種類の選択」の歯車アイコン → **「ウェブアプリ」** を選択
3. 以下の通りに設定する：
   | 項目 | 設定値 |
   |------|--------|
   | 説明 | 良かったこと日記 API（任意） |
   | 次のユーザーとして実行 | **自分** |
   | アクセスできるユーザー | **全員** |
4. **「デプロイ」** をクリック
5. 初回は「権限の確認」が求められます：
   - 「アクセスを承認」→ Googleアカウントを選択
   - 「このアプリはGoogleで確認されていません」と表示される場合  
     →「詳細」→「安全でないページに移動（続行）」をクリック
   - 「許可」をクリック
6. デプロイ完了後に表示される **「ウェブアプリのURL」をコピーしてメモする**

> URL の形式：`https://script.google.com/macros/s/XXXXX.../exec`

---

## STEP 3：フロントエンドにGAS URLを設定する

`script.js` を開き、冒頭の定数を書き換えます：

```javascript
// 変更前
const GAS_URL = "https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec";

// 変更後（STEP 2-3でメモしたURLに書き換える）
const GAS_URL = "https://script.google.com/macros/s/AKfy...実際のID.../exec";
```

---

## STEP 4：GitHub Pages にデプロイする

1. GitHubで新しいリポジトリを作成（例：`yokatta-koto`）
2. 以下の3ファイルをリポジトリにアップロード：
   - `index.html`
   - `style.css`
   - `script.js`（GAS URLを設定済みのもの）
3. リポジトリの **「Settings」→「Pages」** を開く
4. 「Branch」を `main`（または `master`）、フォルダを `/root` に設定して **「Save」**
5. しばらく待つと以下のURLでアクセスできます：  
   `https://【GitHubユーザー名】.github.io/yokatta-koto/`

---

## STEP 5：動作確認

1. GitHub PagesのURLにアクセス
2. 3つのテキストエリアに内容を入力して「保存する」
3. スプレッドシートを開き、`diary` シートにデータが保存されていることを確認
4. ページをリロードして、過去ログに表示されることを確認

---

## トラブルシューティング

### 「通信エラーが発生しました」と表示される

- `script.js` の `GAS_URL` が正しいか確認
- GASのデプロイで「アクセスできるユーザー：全員」になっているか確認
- GASを再デプロイ（「デプロイの管理」→「新しいバージョン」）してみる

### データが保存されない / 取得できない

- ブラウザの開発者ツール（F12）→「Console」でエラーを確認
- GAS側でログを確認：エディタ上で「実行ログ」を開く

### GASを更新した場合

コードを変更したら必ず「新しいバージョン」として再デプロイしてください。  
「デプロイ」→「デプロイを管理」→「編集（鉛筆アイコン）」→「新しいバージョン」

---

## セキュリティについて

- スプレッドシートIDはGAS（サーバー側）にのみ記述しており、フロントエンドには公開されません
- フロントエンドはGAS WebアプリのURLのみを使用します
- GASの「次のユーザーとして実行：自分」の設定により、スプレッドシートへのアクセス権は実行者（あなた）の権限で制御されます

## データ構造（スプレッドシート）

| A列 (id) | B列 (good1) | C列 (good2) | D列 (good3) |
|----------|-------------|-------------|-------------|
| 1749200000000 | コーヒーが美味しかった | 友達に連絡した | 空がきれいだった |
| 1749100000000 | ... | ... | ... |

- `id` にはミリ秒タイムスタンプを使用（一意のキーとして機能）
- 日時表示はフロントエンド側でタイムスタンプから計算します
