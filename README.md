# 今日の良かったこと - セットアップ手順

## 1. スプレッドシートを作成
新しいGoogleスプレッドシートを1つ作成し、そのIDをコピーしておく(URLの `/d/` と `/edit` の間の文字列)。
シート(Users, Entries, Comments, Likes)は初回アクセス時(または`setup()`実行時)にコードが自動作成します。

## 2. Apps Script プロジェクト作成
1. スプレッドシートの「拡張機能」→「Apps Script」を開く
2. `Code.gs` の中身を、このリポジトリの `Code.gs` の内容に置き換える
3. 左メニューの歯車アイコン→「スクリプト プロパティ」で以下を追加:
   - `SPREADSHEET_ID` : 手順1でコピーしたID
   - `PEPPER` : 好きな長いランダム文字列(例: 32文字くらいの英数字。他人に教えないこと)

## 3. 初期化関数を実行(★重要・最初に1回だけ)
1. Apps Scriptエディタ上部の関数選択プルダウンで `setup` を選ぶ
2. ▷実行 をクリック
3. 「承認が必要です」と出たら許可する(スプレッドシート・メール送信の権限)
4. 実行ログに「OK」と出れば完了(Users, Entries シートが自動生成されます)

これをやらずにいきなりウェブアプリとしてデプロイすると、初回アクセス時に権限エラーになることがあります。

※ 以前のバージョンで既にEntriesシートを作成済みの場合、K列(UpdatedAt)が無いことがあります。その場合はK1セルに「UpdatedAt」と手入力しておくと分かりやすいです(無くても動作に支障はありません)。

## 4. デプロイ
1. 右上「デプロイ」→「新しいデプロイ」
2. 種類: ウェブアプリ
3. 実行ユーザー: 自分 / アクセスできるユーザー: 全員
4. デプロイ後に表示される **ウェブアプリURL** をコピー

※ 既にデプロイ済みの場合、Code.gsを更新したら **必ず「新しいバージョン」でデプロイし直す** こと(いつもの注意点)。

## 5. フロントエンド設定
`index.html` 内の

```js
const API_URL = "GAS_WEB_APP_URL_HERE";
```

を、手順3でコピーしたウェブアプリURLに書き換えて、GitHub Pagesリポジトリ `yokatta-koto` に配置する。

## 機能まとめ
- 会員登録: メール+ニックネーム→仮パスワードをメール送信→初回ログイン時に強制パスワード変更
- パスワードを忘れた場合、ログイン画面から仮パスワードを再発行可能(次回ログイン時に強制的にパスワード変更)
- 登録時に注意事項・免責事項への同意が必須
- ログイン画面でパスワードの表示/非表示切り替え可能
- 設定画面からニックネーム・パスワードをいつでも変更可能
- 投稿は「1つ目・2つ目・3つ目」がそれぞれ独立して公開/非公開を選択可能
- 過去の記録は「新しい順」「ランダム表示」「自分の記録」の3タブで閲覧
- 自分の投稿は非公開項目も含めて全て自分には表示され、削除も可能
- 他人の投稿は公開に設定された項目のみ表示(非公開項目はそもそも他人には送信されません)
- 自分の投稿はいつでも「修正」で内容・公開設定を変更可能(削除も可)
- 閲覧できる投稿(自分の投稿、または公開項目がある他人の投稿)にコメント・いいねが可能
- コメントには1階層まで返信可能。コメント・返信にもいいねが可能
- 自分のコメント・返信は修正・削除が可能(親コメントを削除するとその返信もまとめて削除されます)
- いいねはボタンを押すたびにON/OFFが切り替わる方式(押し直すと取り消し)
